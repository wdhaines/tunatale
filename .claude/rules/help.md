# When to Ask for Help

## Proactive Escalation

If you're stuck or going in circles for more than a few attempts:

1. **Stop spinning** — don't keep trying the same failing approach
2. **Ask for advice** from a stronger model (Claude Opus 4.7 with max thinking)
3. **Be specific** — explain what you've tried and why it's not working

## Signs You Should Ask

- Same test failing after 3+ different fix attempts
- Coverage gaps you can't hit after multiple approaches
- Unclear what the correct implementation should be
- Going in circles between two or more failed approaches

## How to Ask

Use the question tool to request guidance:

```
"I'm stuck on [specific problem]. I've tried [approach 1], [approach 2], [approach 3]. 
Can you ask Claude Opus 4.7 with max thinking to help?"
```

Don't waste time — ask early rather than after many failed attempts.
